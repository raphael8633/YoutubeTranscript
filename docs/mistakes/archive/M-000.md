# M-000 Windows Bash tool is Git Bash (MSYS/MINGW64)

**Archived:** 2026-03-23
**Reason:** Project moved to Linux environment — Windows-specific rules no longer applicable.

- Trigger: Using Bash tool on Windows
- Symptom: CMD-style commands (`cd /d`, `dir`, backslash paths) fail in Git Bash
- Guard: Use Unix paths (`/d/dev/...`), `ls`, `mkdir -p`. For `mklink` use `cmd.exe /c "..."`. For `.ps1` scripts use `powershell.exe -ExecutionPolicy Bypass -File script.ps1`
- Verification: Commands execute without "command not found" errors
